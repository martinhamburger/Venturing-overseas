# 决策记录

## D-001：默认采用“规则驱动 + 模板驱动”的海报流程

时间：2026-03-10

决策：

- 不让 Codex 从零自由设计整张海报
- 先固定模板、区块结构和字数边界

原因：

- 研究海报比普通宣传海报更依赖信息层级
- 当前课题内容已经足够明确，重点是压缩与组织，而不是发散生成

## D-002：将 `poster-agent/` 设为可复用中间层

时间：2026-03-10

决策：

- 将任务说明、prompt、规则、模板和输出集中到 `poster-agent/`

原因：

- 后续可能更换 Figma 模板
- 后续可能重审工作流
- 需要把“内容逻辑”和“视觉实现”拆开

## D-003：将 `poster/` 作为本地高保真原型层

时间：2026-03-10

决策：

- 使用 HTML/CSS 原型验证信息密度和阅读路径

原因：

- 本地迭代速度快
- 适合后续通过 `generate_figma_design` 导入 Figma
- 有利于把视觉修改和研究内容修改分开处理

## D-004：Figma MCP 作为桥接层，不作为唯一设计引擎

时间：2026-03-10

决策：

- 默认把 MCP 用在读取设计上下文和导入视觉原型
- 不把“任意写入 Figma 设计对象”视为当前主干流程

原因：

- 官方能力目前更稳定的场景是读取上下文和导入 live UI
- 完整自动排版仍需要人为控制审美和细节

## D-005：当前项目先固定 2024 年数据口径

时间：2026-03-10

决策：

- 暂时以已整理的 2024 年公开行业数据为当前海报的默认事实基线

原因：

- 本地课题文稿已经围绕 2024 年数据整理
- 在没有补充更新证据前，继续追求“最新数字”会让内容口径频繁摇摆

## D-006：项目范围从“光伏单行业”纠正为“总课题综合海报”

时间：2026-03-10

决策：

- 海报不再围绕单一光伏产业
- 改为围绕结题报告总题目，整合日本经验、家电、新能源和综合建议

原因：

- 结题报告的正式标题是《贸易战2.0背景下中国企业出海的应对策略研究》
- 光伏只是新能源章节中的一个子部分
- 如果海报只保留光伏，会偏离结题成果全貌
