# Figma 交接说明

## 1. 这次要交给 Figma 的核心内容

- 标题与作者信息
- 核心结论带
- 研究框架卡片
- 关键数字卡片
- 日本经验面板
- 中国行业实证矩阵
- 跨行业趋势面板
- 政府建议与企业建议

## 2. 推荐在 Figma 中的区块顺序

1. `title/main`
2. `meta/authors`
3. `meta/affiliation`
4. `takeaway/main`
5. `framework/cards`
6. `stats/cards`
7. `section/japan-patterns`
8. `section/china-industry-matrix`
9. `section/cross-case-findings`
10. `section/common-challenges`
11. `section/government-strategy`
12. `section/enterprise-strategy`

## 3. 建议人工补充的内容

- 学校或院系视觉标识
- 答辩场景下的图示与流程箭头
- 最终导出的页边距与打印参数
- 参考文献的最终呈现形式

## 4. MCP 使用建议

### 如果新建 Figma 设计稿

- 使用远程 MCP
- 用 `generate_figma_design` 导入本地页面
- 导入后保留当前版为 `v1`，不要覆盖

### 如果基于已有 Figma 模板

- 先用桌面 MCP 读取母版 frame 的上下文和截图
- 再将本地内容映射回模板结构
- 优先保持模板框架，只替换文字、表格与高层级色彩

## 5. 当前版本的定位

- 当前版本是综合研究海报的内容初稿
- 重点是完整覆盖总课题，而不是做到最终展示精修
- 若进入终稿阶段，下一步应把行业矩阵转成更高质量的 Figma 表格或信息图
