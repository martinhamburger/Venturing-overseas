# Poster Checklist

Use this checklist when a poster request becomes interactive or when you need a fast review pass.

## Input Checklist

Confirm or infer:

- source files and their paths
- poster size, orientation, DPI, export format, and file-size limits
- mandatory sections, authors, advisor, affiliation, logo, and reflection text
- official template, screenshot, or existing Figma frame
- whether the user wants local prototype only, Figma upload, or both
- whether charts are required and which data is approved

## Copy Compression Targets

- title: ideally within 24 Chinese characters
- title: ideally within two lines
- core takeaway: about 40-60 Chinese characters
- ordinary content block: about 90-120 Chinese characters
- outside the abstract, avoid long continuous paragraphs
- do not repeat the same point more than twice on the same poster

## Layout And Typography Checks

- conclusion and key numbers appear before background detail
- reading path is obvious within 3 or 4 columns
- body text stays at poster-readable size
- typography uses 2 font families or fewer unless the user specifies otherwise
- palette stays restrained and high-contrast
- tables are compressed to essential fields only
- whitespace feels intentional
- the page feels complete rather than half-empty

## Full-Page Fill Pass

When the user explicitly wants the page fully filled:

- rebalance column widths before adding text
- enlarge or shrink key cards before adding filler sections
- move repeated explanation into figure notes
- use charts, metrics, callouts, or compact comparison blocks to occupy useful space
- keep visible breathing room; “filled” does not mean “packed”

## Chart Checklist

- chart earns its space by making comparison faster
- title states what is being compared
- unit is visible
- numbers come from approved source material
- visual form is simple enough for HTML preview and Figma capture
- 2-3 short bullets explain why the chart matters

## Figma Split

Good candidates for automation:

- content compression
- section order planning
- local prototype updates
- initial Figma capture from local HTML/CSS

Must be checked manually:

- Chinese font fallback
- exact font sizes and letter spacing
- line breaks and alignment
- number and unit alignment
- chart clarity
- logo, school name, and author details
- export margins and final print settings

## Done Checklist

- factual claims match the approved source
- required sections are present
- duplicate points are removed
- no obvious overflow or dead zones remain
- Figma import status is recorded if Figma was used
- remaining manual polish items are clearly listed
